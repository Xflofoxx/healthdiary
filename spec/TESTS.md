# SPEC Tests

Version: 1.0.0
Last Updated: 2026-02-27

## Test Types
- Unit tests for individual components
- Integration tests for interactions between modules
- Property-based tests for random inputs
- Contract tests for API boundaries
- End-to-end tests for flows

## Coverage Criteria
- Minimum 80% line/branch coverage (adjust to project needs)
- Critical paths fully covered

## Test Artifacts
- Test data located under `tests/data/` or equivalent
- Coverage reports generated and attached to PRs

## How to Validate SPEC Alignment
- Ensure test suites exist and run in CI
- Check that specs (tests.md) reflect current test strategy
